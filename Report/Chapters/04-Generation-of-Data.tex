\chapter[Generation of Data]{Generation of Data}
\label{cp:generation-of-data}

Now that the control flags are defined, we can proceed with the first step of
the simulation: generating the raw binary data. To accurately simulate a
real-world communication source, we use the MATLAB function \texttt{randi} to
produce a random bit-stream.

\begin{listing}[!htpb]
    \caption{Generation of Data.}
    \label{listing:Data-Generation}
    \begin{minted}{matlab}
%% Data Generation
data = randi([0 1], N_realizations, N_bits);
    \end{minted}
\end{listing}

The function \texttt{randi([0 1], $N_{\text{realizations}}$,
    $N_{\text{bits}}$)} generates a matrix of independent random integers. In this
setup, each row represents a unique \textit{realization} of the random process,
while each column represents a \textit{bit} in the sequence.

This matrix structure is fundamental to our analysis; it allows us to treat the
entire project as a discrete ensemble. By performing operations across the
rows, we can calculate ensemble statistics (like the statistical mean), and by
operating across the columns, we can determine time-domain characteristics
(like the time average). Furthermore, using a range of $[0, 1]$ ensures an
equiprobable source where the probability of a "0" or a "1" is exactly $0.5$,
providing a neutral baseline for comparing the efficiency of different line
codes.
